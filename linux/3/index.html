[Guida] Come abilitare i colori per la sintassi nell’editor nano

Chiunque abbia usato Linux, e nello specifico il terminale, avrà sicuramente avuto a che fare almeno una volta con l’editor di testo nano. Sicuramente la maggior parte delle persone non lo usa per progetti con codice di grande dimensione, ma per scrivere script direttamente da linea di comando può tornare molto utile. La cosa (brutta) che possiamo notare quando lo usiamo, è che di default la colorazione della sintassi è disattivata (al contrario, magari, di quanto avviene invece nel nostro IDE di fiducia), perciò vediamo come fare ad abilitarla.

Scriviamo un semplice script in python che stampa “Hello World!” se passiamo qualsiasi cosa come argomento nella shell quando lo invochiamo:


e vediamo che non ci sono colori, come se fosse un semplice file in formato .txt.
Apriamo un terminale ed eseguiamo

$ ls /usr/share/nano

e vedremo una lista di file. Individuiamo quello che fa al caso nostro (in questo caso sarà python.nanorc), e a questo punto modifichiamo il file di configurazione di nano:

$ sudo nano /etc/nanorc

ed aggiungiamo la linea include "/usr/share/nano/python.nanorc" (è possibile che sia già presente ma commentata, in questo caso basta decommentarla).
